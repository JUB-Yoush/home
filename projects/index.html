Web Development

I've been shedding my wordpress roots and learning web-dev proper for a while now.
Web-dev isn't super difficult, but it's quite complicated. Due to the needs of websites changing but still needing to maintain compatibility with legacy systems, building a modern "web-app" requires learning a lot of different pieces working together to craft the finished product.

I've been having fun though. I think web-dev kinda opens your eyes to all the different applications of software development, as all the little pieces that go together can be delved into and expanded on if you're interested (javascript is everywhere dude.)

I've been following the Odin Project, I like it because it's project based and removes a lot of the "there are 100 different things that could do this thing" decision paralysis that comes with web-development. I've been able to make a lot of cool projects going through it so far, and I'm excited to continue.

One of my faovrite web-dev projects so far was my teams submission for Methacks 2023, Pomoduino. It's a pomodoro timer that sends stats about how you study to a database and displays it to a webpage. I was in charge of the software front-end, back-end and Database. I did a great job (imo)

When I make somthing super hecking impressive, I'll be sure to talk about it here.

Godot Games

I make a lot of games in the Godot Engine, It's free, open source, supports 2d and 3d, and is really good. It's my go-to recommendation for people who are looking for a tool to make games with.

I've published a few projects with it, the most ambitious being Project M.T.S., a Jet Set Radio inspired 3d skating platformer. Even though it's extremely goofy and jank, I was doing things that weren't well documented/hadn't ever been done in the engine before, so it really pushed the limits of my problem solving skills my knowledge of the engine

I have another cool thing in the works, and I'll update the page when it's out.
